# Contributing to CampusLink

Thanks for your interest in contributing to CampusLink! This project is built by students, for students, and we welcome contributions of all sizes.

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your change:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**, following the code style already used in the project
4. **Test your changes** locally before submitting
5. **Commit** with a clear, descriptive message:
   ```bash
   git commit -m "Add: short description of what you did"
   ```
6. **Push** your branch and open a **Pull Request** against `main`

## Commit Message Conventions

- `Add:` for new features
- `Fix:` for bug fixes
- `Update:` for changes to existing functionality
- `Docs:` for documentation-only changes
- `Refactor:` for code changes that don't change behavior

## Reporting Issues

Found a bug or have a feature idea? Open an [Issue](../../issues) with:
- A clear title
- Steps to reproduce (for bugs) or a description of the feature
- Screenshots if relevant

## Code Style

- Follow standard C# naming conventions (PascalCase for public members, camelCase for locals)
- Keep Blazor components small and focused
- Comment non-obvious logic

## Good First Issues

New contributors should look for issues labeled `good first issue` — these are scoped to be approachable for first-time contributors.

## Code of Conduct

Be respectful, constructive, and welcoming. CampusLink is meant to bring students together — contributions should reflect that spirit.
